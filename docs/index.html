You will need the following sofwares to customize this template.

  3. MySQL (for database requirements) (You can use XAAMP: Download)

How to run QuickMailer

Navigate to the project directory and install the required dependencies using npm.

Install dependencies

Start the application

How to Configure QuickMail

Configure the .env file

  • APP_PORT This variable defines the port number on which your application will run. The default value here is 3300. If this port is already in use by another application, you can change it to any other available port.
  • DB_HOST This variable specifies the hostname or IP address of your database server. If your database is running on the same machine as your application, use localhost. Otherwise, specify the IP address or hostname of the remote database server.
  • DB_PORT This variable specifies the port number on which your database server is listening. The default MySQL port is 3306.
  • DB_USERNAME This variable specifies the username for connecting to your MySQL database. Replace root with your actual database username.
  • DB_PASSWORD This variable specifies the password for connecting to your MySQL database. If your database user does not have a password, you can leave this empty. Otherwise, provide the password.
  • DB_DATABASE This variable specifies the name of the database that your application will use. Ensure that this database exists on your MySQL server.
  • POPULATE_DEMO_DATA This variable controls whether the application should populate the database with demo data upon initial setup. Setting this to true can help you get started quickly by providing sample data.
  • INSERT_EMAIL_LOGS This variable determines whether the application should log sent emails into the database. This is useful for tracking and debugging purposes. Set it to true to enable email logging.

    Introduction to QuickMailer

    Settings > Manage Servers

    Before starting to use QuickMail, it is essential to add the SMTP servers that will be used to send emails. The SMTP servers handle the delivery of your emails, and configuring them correctly ensures that your emails are sent reliably and securely.

    Settings > Manage Email Templates

    Managing email templates in QuickMail allows you to create, edit, and organize the templates that you will use to send emails. Setting up templates correctly ensures that your emails are consistent and professional.

    🚀 You are Ready to Send Emails

    After submitting your SMTP server configuration and creating your email templates, you are now ready to send emails with QuickMail. Follow these final steps to start reaching out to your prospects:

    1. Compose New Email:

      Navigate to the Send New Email section from the dashboard. Fill in the receiver's email address, select the template you want to use, and choose the SMTP server from which to send the email.

    2. Send the Email:

      After composing your email, click the Send Email button. Your email will be sent using the specified SMTP server and template.

    By following these steps, you can efficiently manage your email campaigns and contact your prospects with ease. Happy emailing!

    How to manage servers

    Introduction to Managing Servers Managing servers in QuickMail is essential for configuring and maintaining your SMTP settings effectively. This section allows you to add, edit, or delete SMTP servers used for sending emails. Proper server management ensures reliable and secure email delivery, tailored to your specific needs and preferences.

    Add New Server

    • Step 1 - Enter a friendly name for your server. This helps you identify it easily.
    • Step 2 - Provide the hostname or IP address of your SMTP server.
    • Step 3 - Enter the username used for authentication with your SMTP server.
    • Step 4 - Enter the password associated with the SMTP username for authentication.
    • Step 5 - Specify the email address that will appear in the "From" field of your sent emails.
    • Step 6 - Enter the port number used by your SMTP server for sending emails.
    • Step 7 - Click the submit button to save your server configuration.
      Now you are ready to create your first template and send email

    Remove Servers -

    Removing servers in QuickMail allows you to manage your SMTP configurations efficiently. Whether you need to replace outdated servers or streamline your setup, this process ensures your email delivery remains optimized.

    • Step 1 - Navigate to Manage Servers from the Settings section.
    • Step 2 - Scroll to the bottom of the screen.
    • Step 3 - Click on the Delete button next to the server you wish to remove.

    Add New Template

    When adding a new template in QuickMail, you're creating a predefined format for your emails. Templates simplify the process of composing and sending emails by allowing you to quickly select the appropriate template when drafting a new message. This ensures consistency and saves time, especially when sending frequent communications to your contacts.

    • Template Name: Give your template a descriptive name.
    • Template Subject: Enter the subject line for your email.
    • Template Content: Craft the main message of your email.
    • Flag template content as HTML Type: Check this box if your template includes HTML formatting.

    After filling out the template details, click the submit button to save your new email template.
    Now you are ready to create your first e-mail and send emails using that template.

    How to send emails

    To send emails in QuickMail, you must have at least one SMTP server and one email template configured beforehand. This ensures seamless delivery and consistency in your communications.

    Composing Email Form Requirements

    • Receiver(s) Email - Enter the email address(es) of the recipient(s). Separate multiple emails with commas if sending to multiple recipients.
    • Template - Select the email template you wish to use from the dropdown menu. Templates streamline the content of your emails, making it easier to send consistent messages.
    • Server - Choose the SMTP server from which you want to send the email. This ensures that your emails are sent using the correct configuration and from the designated email address.
